# Локализация (kk / ru / en)

Atlas поддерживает три языка в интерфейсе и в сообщениях API.

## Поддерживаемые локали

| Код | Язык |
|-----|------|
| `en` | English (fallback) |
| `ru` | Русский |
| `kk` | Қазақша |

## Frontend (next-intl)

| Файл | Назначение |
|------|------------|
| `frontend/src/i18n/routing.ts` | Список локалей, default |
| `frontend/src/i18n/request.ts` | Загрузка messages |
| `frontend/middleware.ts` | Cookie `NEXT_LOCALE`, детекция |
| `frontend/src/app/layout.tsx` | `NextIntlClientProvider` |
| `frontend/messages/{en,ru,kk}.json` | Каталоги строк |

### Переключатель языка

`LanguageSwitcher.tsx` сохраняет:

- cookie `NEXT_LOCALE`
- `localStorage` ключ `atlas_locale`

### Использование в компонентах

```tsx
import {useTranslations} from "next-intl";

const t = useTranslations("schedule");
return <h1>{t("title")}</h1>;
```

### Запросы к API

Клиент отправляет:

```http
X-Locale: ru
Accept-Language: ru,en;q=0.9
```

Реализация — в `frontend/src/lib/api.ts`.

## Backend

| Файл | Назначение |
|------|------------|
| `backend/app/i18n/core.py` | Каталоги, `t()`, `resolve_locale()` |
| `backend/app/main.py` | Middleware + обработчики ошибок |

### Приоритет локали

1. Заголовок `X-Locale`
2. `Accept-Language`
3. `en`

Ответ содержит `Content-Language: <locale>`.

### Ошибки HTTP

```python
raise HTTPException(
    status_code=403,
    detail={"key": "errors.insufficientPermissions"},
)
```

Клиент получает:

```json
{
  "detail": "Недостаточно прав",
  "code": "errors.insufficientPermissions"
}
```

### Ошибки валидации расписания

Стабильные коды (`TEACHER_DOUBLE_BOOKING`, …) локализуются через `localize_issue(...)` в validation flow. Ключи в каталогах `validation.*`.

## Как добавить новую строку

### UI

1. Добавить ключ в **все три** файла `frontend/messages/*.json`.
2. Использовать `useTranslations("namespace")` в компоненте.

Соглашение по префиксам:

| Префикс | Область |
|---------|---------|
| `nav.*` | Навигация |
| `pages.*` | Заголовки страниц |
| `crud.*` | CRUD-панели |
| `schedule.*` | Редактор расписания |
| `errors.*` | Ошибки (частично дублирует API) |
| `validation.*` | Коды конфликтов |

### API error

1. `detail={"key": "errors.myKey", "params": {...}}` в backend.
2. Строка в `backend/app/i18n/core.py` для `en`, `ru`, `kk`.

### Validation issue

1. Новый `code` в `validation_engine.py` + `constraint_catalog.py`.
2. Перевод в backend i18n и `validation.*` во frontend messages.

## Проверка

- Переключить язык в UI → перезагрузить страницу.
- Вызвать API с `X-Locale: kk` и проверить `detail` на казахском.
- Убедиться, что не осталось «сырых» ключей вида `schedule.unknownKey`.

## Заметки

- Не переименовывайте существующие ключи без массового обновления JSON.
- Параметры в строках: `{name}`, `{count}` — через `t("key", { name: "..." })`.
- Даты и числа форматируйте через `Intl` при необходимости (отдельно от next-intl).
